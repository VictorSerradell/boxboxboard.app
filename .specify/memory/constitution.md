# Simplan Constitution

## Core Principles

### 1. Simplicity First

Siempre priorizar la solución más simple que funcione (YAGNI, KISS). Evitar abstracciones prematuras y over-engineering.

### 2. Code Quality

Código limpio, legible y bien comentado. Seguir PEP 8 (o el estándar que uses). Nombres de variables claros. Funciones cortas y con responsabilidad única.

### 3. Testing

Todo código nuevo debe tener tests. Preferiblemente tests unitarios + tests de integración cuando sea relevante. No mergear código sin tests que pasen.

### 4. Documentation

Mantener README actualizado y documentar funciones y módulos importantes. Todo comportamiento no obvio debe estar documentado.

### 5. Performance Aware

(Si tu proyecto es de simulación o planificación) Optimizar cuando sea necesario, pero solo después de medir. Evitar código lento en loops principales.
